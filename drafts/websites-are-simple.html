Websites are Simple!

Posted on Tue 21 January 2025 in Post

As a beginner learning how websites work, it is very easy for it to become too complicated. Especially for web developers looking for a quick entry into a job. For myself, the best was to learn web development comes from a different place.
Learning to Hack Websites

Intro: Learning C vs Python

When learning how technology works, it is crucial to actually learn how the technology works. Layers of abstractions can be helpful when developing a distributed data analysis platform or building scam baiting chatbots, but hiding what is going on does not help someone learn what is going on. The problem is the magic. If a process is magic and just works, then a beginner will not learn what is going on.

Sometimes abstractions are good. While I love assembly code and thinking about stack pointers, it is not important to most modern programmers (especially when developing for different computers with different architectures and assembly code versions). When writing print("Hello World!") or printf("Hello World!\n"); it is not important to understand how this magically goes to the terminal output and not the printer, but it could be useful to understand the difference between the source code and the compiled code. This is where there is a clear difference in learning to program as a beginner in the C language and Python.

Hello World in Python contains 2 new things for a beginner, calling a function print () and a string "Hello World". As a beginner this is simple and can be done quickly. Compare this to Hello World in C which requires importing the standard library, defining a 'main' function and putting code inside of it (hopefully returning 0). Then when you can finally write the print statement, you have to misspell print with an f, and a \n that looks like a different colour in the code editor, and finally put a ; at the end (once you find where that key is for the first time ever).
